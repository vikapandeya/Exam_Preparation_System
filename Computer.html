Section 1

Introduction to Computers

From the first computing machines to modern AI-powered devices — understand what computers are and how they evolved.

What is a Computer?

A computer is an electronic device that processes data according to a set of instructions (programs) to produce useful output. It can accept data (input), process it, store it, and produce results (output).

Key Definition: Computer = Input + Processing + Storage + Output
  • Accepts data through input devices
  • Processes data using CPU
  • Stores data in memory and storage
  • Produces output via output devices
  • Follows precise programmatic instructions

1st Generation

Used vacuum tubes. Examples: ENIAC, UNIVAC. Huge, expensive, produced lots of heat. Machine language programming.

2nd Generation

Transistors replaced vacuum tubes. Smaller, faster, cheaper. Assembly language introduced. IBM 7090, UNIVAC 1107.

3rd Generation

Integrated Circuits (ICs). Multiple transistors on a chip. High-level languages (COBOL, FORTRAN). IBM 360 series.

1971–Present

4th Generation

Microprocessors (VLSI). Personal computers, GUIs, networks. Intel 4004 → Core i9. Windows, macOS, Linux.

Future

5th Generation

AI, machine learning, quantum computing, natural language processing. Voice assistants, self-driving systems.

Fastest computers. Used for scientific simulations, weather forecasting, nuclear research. Example: Frontier (Oak Ridge National Lab) — 1.102 ExaFLOPS.

Most common personal computer. Sits on/under a desk. Upgradeable, powerful, best value for performance per dollar.

Smartphone

HDD (Hard Disk Drive)

Magnetic spinning platters. 500GB–20TB. 5400–7200 RPM. Slower (80–160 MB/s) but cheap per GB. Good for bulk storage and backups.

SSD (Solid State Drive)

Flash memory, no moving parts. 120GB–8TB. 400–550 MB/s (SATA). Silent, durable, 5–10× faster than HDD. Ideal for OS and apps.

NVMe SSD

PCIe interface SSD. 3,500–7,000 MB/s (Gen 4). M.2 form factor. Fastest consumer storage available. Best for intensive workloads.

External Drives

Portable storage via USB 3.0/USB-C. HDD (bulkier, cheaper) or SSD (compact, faster). Ideal for backups and file transfers.

Optical Drives

CD, DVD, Blu-ray. Mostly legacy now. BD-ROM holds 25–100GB. Still used for software distribution and archiving.

Cloud Storage

Google Drive, OneDrive, iCloud, Dropbox. Accessible anywhere. Requires internet. Ideal for sync and collaboration.

Graphics Processing Unit (GPU)

Handles visual rendering and parallel computations. Originally for games — now essential for AI, machine learning, and scientific computing.

CPU vs GPU: CPU has 8–64 powerful cores optimized for sequential tasks. GPU has thousands of smaller cores optimized for parallel tasks (rendering millions of pixels simultaneously).

Section 7

Cybersecurity

Protect systems, data, and privacy from digital threats.

Virus

Self-replicating code attached to programs. Spreads via files, email, USB. Can corrupt/delete data.

Worm

Self-replicates across networks without user action. WannaCry hit 200K computers in 150 countries.

Trojan

Disguised as legitimate software. Creates backdoor. Does not self-replicate but causes major damage.

Ransomware

Encrypts your files, demands crypto payment. Avg demand: $1.54M (2023). Backup is your best defense.

Spyware

Silently monitors — keylogging, screenshots, steals credentials. Often bundled with free software.

Phishing

Fake emails/sites steal credentials. 91% of attacks start here. Check sender domain carefully.

Defense Tools

Firewalls
Stateful packet inspection. NGFW adds deep packet inspection, IDS/IPS. Hardware at router + software on OS level.
Multi-Factor Authentication (MFA)
Something you KNOW + HAVE + ARE. Blocks 99.9% of account compromises (Microsoft data). Enable on email, banking, social accounts.
VPN
Encrypts all traffic in a tunnel. Hides your real IP. Essential on public WiFi. Protocols: WireGuard, OpenVPN, IKEv2.
Encryption
AES-256 symmetric for bulk data. RSA/ECC asymmetric for key exchange (HTTPS). SHA-256 hashing for passwords (always salt+hash, never plain).

Warnings & Red Flags

Phishing: Urgent language, mismatched domains, generic greetings, requests for passwords or payment info.
Malware: Slow performance, unexpected pop-ups, browser redirects, antivirus suddenly disabled, unknown processes.
Ransomware: Files renamed with random extensions. DISCONNECT from network immediately, restore from backup — do NOT pay.

Personal Security Checklist

  • Unique strong passwords (16+ chars) for each account
  • Enable 2FA on email, banking, social media
  • Keep OS and apps updated — patches fix vulnerabilities
  • Use VPN on public WiFi networks
  • Back up data using 3-2-1 rule
  • Verify sender domain before clicking links
  • Always check HTTPS before logging in
  • Use a password manager (Bitwarden, 1Password)
  • Lock computer when away (Win+L / Cmd+Ctrl+Q)
  • Review app permissions periodically

Core Concepts

Variables & Data Types
int (integers), float (decimals), string (text), boolean (true/false), array/list (ordered collection), dict/object (key-value pairs). Variables are named memory containers.
Control Flow
if/else: conditional. for: fixed iterations. while: condition-based loop. switch/match: multiple cases. break/continue: loop control.
OOP Concepts
Encapsulation: bundle data+methods. Inheritance: child extends parent class. Polymorphism: same method, different behavior. Abstraction: hide complexity, expose interface.
pseudocode// Find largest of 3 numbers INPUT a, b, c IF a > b AND a > c largest = a ELSE IF b > c largest = b ELSE largest = c OUTPUT largest
Big O Notation: O(1) constant · O(log n) log · O(n) linear · O(n²) quadratic. Measures algorithm efficiency.

Python Interview Q&A

List vs Tuple vs Dict vs Set
List [ ] ordered mutable. Tuple ( ) ordered immutable. Dict { } key-value mutable. Set { } unordered no duplicates.
What is a decorator?
A function that wraps another to add behavior. @staticmethod, @classmethod, @property, @app.route. Enables metaprogramming cleanly.
GIL explained
Global Interpreter Lock — one thread runs Python bytecode at a time. Use multiprocessing for CPU-bound, threading for I/O-bound tasks.

Event Loop
Single-threaded. Processes call stack → Web APIs → microtask queue (Promises) → callback queue. Non-blocking I/O via this model.
Closures
Function that remembers its outer scope after the outer function returns. Used for data privacy and factory functions.
var vs let vs const
var: function-scoped, hoisted, legacy. let: block-scoped, re-assignable. const: block-scoped, immutable binding. Always prefer const/let.

Version Control

Git & GitHub

Track changes, collaborate, and manage code history — the #1 developer skill.

What is Git?

Git is a free, distributed version control system created by Linus Torvalds in 2005. It tracks changes to files over time, allowing you to recall any version, collaborate with others, and work on features in isolation.

Git vs GitHub: Git is the tool (runs locally). GitHub is a cloud hosting service for Git repositories. GitLab and Bitbucket are alternatives.
  • Every developer has a full local copy of the repo
  • All changes tracked with timestamp and author
  • Supports non-linear development (branching)
  • Fast — most operations happen offline
  • Industry standard — used by 94% of developers

Branching & Merging

Branches let you work on features/fixes in isolation without affecting the main codebase. The default branch is main (formerly master).

bash# Create and switch to branch git branch feature/login # create git checkout feature/login # switch git checkout -b feature/x # create+switch git switch -c feature/x # modern way # Merge feature into main git checkout main git merge feature/login # merge git branch -d feature/login # delete branch # Rebase (cleaner history) git rebase main # replay on main git cherry-pick abc1234 # pick one commit

Git Workflows

Git Flow
Branches: main (production) · develop (integration) · feature/* (new features) · release/* (prepare release) · hotfix/* (urgent fixes). Complex but structured — great for versioned software.
GitHub Flow
Simple: Create branch → commit changes → open Pull Request → review → merge to main → deploy. Used by GitHub, most web startups. Short-lived branches, continuous deployment.
Trunk-Based Development
All devs commit directly to main (or very short-lived branches). Feature flags hide incomplete features. Requires strong CI/CD and test coverage. Fastest delivery cycle.
Merge Conflicts: Occur when two branches modify the same lines. Git marks conflicts with <<<<<<< ======= >>>>>>>. Resolve manually, then git add + git commit.

Key Concepts

ACID Properties
Atomicity: All or nothing. Consistency: Valid state always. Isolation: Concurrent transactions don't interfere. Durability: Committed data persists even after crash.
Primary Key vs Foreign Key
Primary Key: Uniquely identifies each row in a table. Cannot be NULL. Foreign Key: Column referencing the Primary Key of another table. Enforces referential integrity.
Database Normalization
Process of organizing a database to reduce redundancy. 1NF: atomic values. 2NF: no partial dependencies. 3NF: no transitive dependencies. BCNF: stronger version of 3NF.

Computer Maintenance & Troubleshooting

Keep your computer fast, clean, and healthy — and fix problems when they arise.

Physical Cleaning

Clean dust from vents, fans, heatsinks every 6 months. Use compressed air. Dust buildup = overheating = throttling. Wipe screen with microfibre cloth only.

Disk Cleanup

Run Disk Cleanup (Windows) — delete temp files, recycle bin, update cache. Use CCleaner or BleachBit. Free up space to maintain SSD performance.

Software Updates

Keep OS, drivers, and software updated. Patches fix security vulnerabilities. Enable Windows Update, check for GPU/motherboard BIOS updates quarterly.

Temperature Monitoring

CPU: safe 60-80°C under load. GPU: safe 70-85°C. Use HWiNFO, Core Temp, or GPU-Z to monitor. Reapply thermal paste every 2-3 years.

Malware Scanning

Run weekly antivirus scans. Windows Defender (built-in) + Malwarebytes (second opinion). Don't run two real-time AVs simultaneously — conflict!

Power Management

Use UPS (Uninterruptible Power Supply) to protect against surges. For laptops: 20-80% charge rule preserves battery health long-term.

Common Problems & Fixes

Slow PC
1. Check Task Manager for high CPU/RAM processes. 2. Disable startup programs. 3. Run Disk Cleanup. 4. Scan for malware. 5. Upgrade to SSD. 6. Add more RAM. 7. Check disk health (crystaldiskinfo).
No Display / Black Screen
1. Check monitor cable and power. 2. Try different video port (HDMI/DP). 3. Reseat RAM sticks. 4. Test with one RAM stick. 5. Reset CMOS (remove motherboard battery 30 sec). 6. Reseat GPU. 7. Test with integrated graphics.
Internet Not Working
1. Restart router and modem (wait 30s). 2. ipconfig /release then /renew. 3. netsh winsock reset. 4. ipconfig /flushdns. 5. Update network drivers. 6. Check if other devices work (ISP issue vs device issue).
Blue Screen of Death (BSOD)
1. Note the error code (e.g., CRITICAL_PROCESS_DIED). 2. Run sfc /scannow. 3. Run DISM /Online /Cleanup-Image /RestoreHealth. 4. Check Event Viewer → Windows Logs → System. 5. Update or rollback recent drivers. 6. Run Windows Memory Diagnostic.
No Sound
1. Check volume mixer (right-click speaker icon). 2. Run audio troubleshooter. 3. Set correct default playback device. 4. Update audio drivers. 5. Check 3.5mm plug is in correct jack (green = output, pink = mic).
Overheating / Shutting Down
1. Clean dust from all vents and fans. 2. Check thermal paste — reapply if dry. 3. Ensure good airflow in case (front intake, rear/top exhaust). 4. Set Power Plan to Balanced (not High Performance). 5. Use HWiNFO to identify hottest component.

Section 18

Interview Preparation

Top questions asked in computer science, IT, and software engineering interviews.

What is the difference between RAM and ROM?
RAM (Random Access Memory) is volatile — it loses data when power is off. It's fast and holds currently running programs. ROM (Read-Only Memory) is non-volatile — retains data without power. Used for firmware/BIOS. Modern systems use Flash memory (SSD/eMMC) as a ROM replacement.
What is an operating system? Name its functions.
An OS manages hardware and software resources. Functions: Process Management (CPU scheduling), Memory Management (RAM allocation), File System Management, Device Management (drivers), Security, and User Interface (CLI/GUI).
Explain the difference between 32-bit and 64-bit OS.
32-bit OS can address max 4GB RAM (2³²). 64-bit OS addresses up to 18 exabytes theoretically. 64-bit runs 32-bit apps via compatibility mode but not vice versa. Modern computers are all 64-bit. 64-bit also improves CPU register width and arithmetic precision.
What is virtual memory?
Virtual memory uses disk space (pagefile/swapfile) as an extension of RAM when physical RAM is full. Slower than RAM (100-1000× slower on HDD). The OS swaps least-used pages to disk. On SSD it's more tolerable. Avoid relying on it — add more physical RAM instead.
What is BIOS and what does POST stand for?
BIOS (Basic Input/Output System) / UEFI is firmware that initializes hardware at startup. POST (Power-On Self-Test) runs automatically — it tests RAM, CPU, storage, GPU before loading the OS. Beep codes indicate hardware failures during POST.
What is the difference between process and thread?
A process is an independent program in execution with its own memory space. A thread is the smallest unit of execution within a process — threads share the process's memory. Context switching between processes is more expensive than between threads. Multi-threading enables parallelism within one app.
Explain cache memory levels (L1, L2, L3).
L1: Fastest, smallest (32–128KB), per-core. ~4 CPU cycles latency. L2: Larger (256KB–2MB), per-core, slightly slower. L3: Shared across all cores (8–96MB), ~40 cycles. Data is checked L1 → L2 → L3 → RAM → Storage (slowest). Cache hit vs cache miss greatly impacts CPU performance.
What is the difference between compiler and interpreter?
Compiler translates entire source code to machine code at once before execution (C, C++, Go, Rust). Interpreter executes code line by line at runtime (Python, Ruby). JIT (Just-In-Time) compilers like in Java and JavaScript combine both — compile at runtime for performance.
What is an IP address and why do we need it?
An IP address uniquely identifies a device on a network. IPv4: 32-bit (192.168.1.1), ~4.3 billion addresses. IPv6: 128-bit (2001:db8::1), virtually unlimited. Needed for routing: routers use IP addresses to direct packets to the correct destination across the internet.
What is the difference between TCP and UDP?
TCP: connection-oriented (3-way handshake), reliable delivery, ordered, error-checked. Used for HTTP, email, file transfer. UDP: connectionless, no guarantee of delivery or order, very fast. Used for video streaming, gaming, DNS, VoIP where speed > reliability.
What is subnetting?
Dividing a network into smaller sub-networks. A /24 subnet has 256 addresses (254 usable). CIDR notation: 192.168.1.0/24. Subnet mask 255.255.255.0 means first 3 octets = network, last = host. Benefits: reduces broadcast traffic, improves security, better IP management.
What happens when you type google.com in a browser?
1. Browser checks local DNS cache. 2. DNS resolver queries Root → TLD → Authoritative NS. 3. Gets IP address. 4. TCP 3-way handshake with server. 5. TLS handshake (HTTPS). 6. HTTP GET request sent. 7. Server returns HTML. 8. Browser parses HTML, fetches CSS/JS, renders page.
What is NAT and why is it used?
NAT (Network Address Translation) maps private IPs (192.168.x.x) to a single public IP. Conserves public IPv4 addresses. Your home router uses NAT — all your devices share one public IP. Port Address Translation (PAT) tracks connections by port number to route return traffic correctly.
What is a firewall and how does it work?
A firewall monitors and controls network traffic based on security rules. Packet filtering checks IP/port. Stateful inspection tracks active connections. Application layer (Layer 7) inspects content. NGFW adds IDS/IPS, SSL inspection, and threat intelligence. Can be hardware (router) or software (Windows Firewall).
Difference between HTTP and HTTPS?
HTTP (port 80) is unencrypted — data sent in plaintext, vulnerable to interception. HTTPS (port 443) adds TLS encryption — all data is encrypted end-to-end. TLS uses asymmetric encryption (RSA/ECC) for key exchange, then symmetric (AES-256) for data. Always use HTTPS on login pages and anywhere sensitive data is transmitted.
What is the difference between stack and heap memory?
Stack: automatic memory for local variables and function calls. LIFO structure. Fast, fixed size, managed by OS. Heap: dynamic memory allocated at runtime (malloc/new). Larger but slower. Developer must manage in C/C++ (free/delete). Garbage collected in Java/Python.
What is Big O notation? Give examples.
Describes algorithm efficiency as input n grows. O(1) constant: array index. O(log n) logarithmic: binary search. O(n) linear: simple loop. O(n log n): merge sort, quick sort. O(n²) quadratic: nested loops, bubble sort. O(2ⁿ) exponential: recursive Fibonacci. Always aim for lowest complexity.
What are the 4 OOP principles?
Encapsulation: bundle data + methods, hide internals. Inheritance: child class inherits parent's properties/methods. Polymorphism: one interface, multiple implementations (method overriding/overloading). Abstraction: hide complexity, expose only necessary interface via abstract classes/interfaces.
What is recursion? Give an example.
A function calling itself with a smaller problem until a base case is reached. Example: factorial(5) = 5 × factorial(4) = ... = 5×4×3×2×1. Base case: factorial(0) = 1. Risk: stack overflow if base case missing. Used in trees, graphs, divide-and-conquer algorithms.
What is a RESTful API?
REST (Representational State Transfer) is an architectural style for APIs. Uses HTTP methods: GET (read), POST (create), PUT/PATCH (update), DELETE. Stateless — each request is independent. Returns JSON/XML. Status codes: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Server Error.
Explain Git branching strategy.
Git Flow: main (production), develop (integration), feature/*, release/*, hotfix/* branches. Trunk-Based Development: short-lived feature branches merged to main daily. GitHub Flow: feature branches + pull requests to main. git commit/push/pull/merge/rebase are core operations.
What is SQL injection and how to prevent it?
SQL injection inserts malicious SQL via user input. Example: entering ' OR 1=1 -- in a login form bypasses authentication. Prevention: use parameterized queries / prepared statements, ORM frameworks, input validation, least-privilege DB accounts, WAF (Web Application Firewall).
What is XSS (Cross-Site Scripting)?
Attacker injects malicious JavaScript into web pages viewed by other users. Stored XSS: saved in database. Reflected XSS: in URL parameter. Prevention: sanitize/escape all user input, Content Security Policy (CSP) headers, HTTPOnly cookies, avoid innerHTML with user data.
Explain symmetric vs asymmetric encryption.
Symmetric: same key for encrypt and decrypt (AES-256). Fast, used for bulk data. Key distribution is the challenge. Asymmetric: public key encrypts, private key decrypts (RSA, ECC). Solves key distribution. Slower. HTTPS uses asymmetric for key exchange then symmetric for data transfer.
What is the principle of least privilege?
Users and programs should have only the minimum permissions needed to perform their function. Reduces attack surface. A compromised low-privilege account can cause less damage. Examples: DB user with SELECT only (not DROP), app running as non-root, employees can only access their own department's data.
What is a zero-day vulnerability?
A software vulnerability unknown to the vendor — zero days to fix. Attackers exploit it before a patch exists. Defense: defense-in-depth (multiple security layers), behavior-based detection (not signature), rapid patch deployment, network segmentation to contain breaches.
What is penetration testing?
Authorized simulated attack on a system to find vulnerabilities before real attackers do. Phases: Reconnaissance → Scanning → Exploitation → Post-exploitation → Reporting. Types: Black box (no info), White box (full info), Grey box (partial). Tools: Nmap, Metasploit, Burp Suite, Wireshark.

Maximum data transfer rate of a network connection, measured in Mbps or Gbps.
BIOS/UEFI
Firmware that initializes hardware during boot before loading the OS.
Bit
Smallest unit of data — either 0 or 1. Eight bits = one byte.
Blockchain
Distributed ledger technology. Immutable chain of records secured by cryptography.
Boolean
Data type with only two values: true or false. Named after mathematician George Boole.
Bootloader
Program that loads the operating system into memory during startup.
Browser
Software for accessing the internet. Chrome, Firefox, Edge, Safari are popular browsers.
Buffer
Temporary memory area used to hold data while it is being transferred or processed.
Bug
Error or flaw in software code that causes unexpected or incorrect behavior.
Byte
Unit of digital data — 8 bits. KB=1024 bytes, MB=1024 KB, GB=1024 MB.
Cache
Fast memory layer that stores frequently accessed data to speed up retrieval.
CLI
Command Line Interface — text-based interface for interacting with computers.
Cloud Computing
Delivery of computing services over the internet — servers, storage, databases, software.
Compiler
Translates high-level source code to machine code all at once. Contrast with interpreter.
Cookie
Small data file stored by browsers to remember user preferences, sessions, and tracking.
CPU
Central Processing Unit — the "brain" of a computer that executes instructions.
Daemon
Background process running continuously on Unix/Linux systems (e.g., httpd, sshd).
Database
Organized collection of structured data stored electronically in a computer system.
Debugging
Process of finding and fixing errors (bugs) in software code.
DNS
Domain Name System — translates domain names to IP addresses.
Docker
Platform for containerizing applications — packages code and dependencies together.
Domain
Human-readable address for a website (e.g., google.com). Registered through registrars.
Driver
Software that allows the OS to communicate with hardware devices.
Encryption
Converting data into unreadable form using algorithms. Requires key to decrypt.
Ethernet
Wired networking technology for LANs. Speeds from 10 Mbps to 400 Gbps.
Firewall
Network security device that monitors and controls incoming/outgoing traffic.
Framework
Pre-built code structure that provides foundation for building applications.
FTP
File Transfer Protocol — standard for transferring files between computers over a network.
GPU
Graphics Processing Unit — processor optimized for parallel computations and rendering.
GUI
Graphical User Interface — visual interface with icons, windows, menus (vs CLI).
Hashing
One-way function converting data to fixed-length string. Used for passwords and integrity checks.
HTML
HyperText Markup Language — standard language for creating web page structure.
HTTP/HTTPS
Protocol for transferring web pages. HTTPS adds TLS encryption for security.
IDE
Integrated Development Environment — tool for writing, testing, and debugging code.
IP Address
Numerical label identifying a device on a network. IPv4 (32-bit) or IPv6 (128-bit).
JSON
JavaScript Object Notation — lightweight data interchange format. Human-readable key-value pairs.
Kernel
Core of an OS that manages hardware resources and provides services to processes.
Kubernetes
Container orchestration platform. Manages deployment, scaling, and networking of containers.
Latency
Time delay between sending a request and receiving a response. Lower is better.
Library
Collection of pre-written code functions/modules that programmers can use in their projects.
Machine Learning
AI subset where computers learn from data without being explicitly programmed.
Malware
Malicious software designed to damage, disrupt, or gain unauthorized access to systems.
Microservices
Architectural approach where applications are built as small independent services.
Middleware
Software connecting different applications or services. The "glue" of enterprise systems.
NAT
Network Address Translation — maps private IPs to a public IP for internet access.
NVMe
Non-Volatile Memory Express — high-speed storage protocol for SSDs via PCIe bus.
OOP
Object-Oriented Programming — paradigm using objects with properties and methods.
Open Source
Software with publicly available source code that anyone can view, modify, and distribute.
OS
Operating System — software managing hardware and providing platform for applications.
Packet
Unit of data transmitted over a network. Contains header (routing info) and payload (data).
Pipeline
Series of processing stages where output of one becomes input of next. CI/CD pipelines automate builds.
Pointer
Variable that stores the memory address of another variable. Used in C/C++.
Protocol
Set of rules governing communication between systems. HTTP, TCP/IP are protocols.
Proxy Server
Intermediary server between clients and destination. Provides caching, filtering, anonymity.
RAM
Random Access Memory — fast volatile memory holding currently active data and programs.
Recursion
Function calling itself. Must have a base case to terminate. Used in trees, fractals, sorting.
Repository
Storage location for software code, typically with version history (e.g., GitHub repo).
REST API
Architectural style for web APIs using HTTP methods (GET, POST, PUT, DELETE).
Router
Networking device that forwards data packets between networks.
SDK
Software Development Kit — tools, libraries, and docs for building on a platform.
Server
Computer providing services to other computers (clients) over a network.
SQL
Structured Query Language for managing relational databases. SELECT, INSERT, UPDATE, DELETE.
SSD
Solid State Drive — flash-based storage. Faster, quieter, and more durable than HDDs.
SSL/TLS
Cryptographic protocols providing secure communication over a network. Powers HTTPS.
Stack
LIFO data structure. Also refers to technology stack (e.g., MEAN, LAMP).
Subnet
Logical subdivision of an IP network. Subnet mask defines network vs host portions.
TCP/IP
Foundation of internet communication. TCP ensures reliable delivery; IP handles routing.
Thread
Smallest unit of program execution. Multiple threads share process memory for parallelism.
URL
Uniform Resource Locator — web address specifying protocol, domain, path, and query.
Variable
Named storage location in memory that holds a value which can change during execution.
Version Control
System tracking file changes over time. Git is the most popular. GitHub/GitLab host repos.
Virtual Machine
Software emulation of a physical computer. VMware, VirtualBox, Hyper-V.
VPN
Virtual Private Network — encrypted tunnel for secure communication over public networks.
WebSocket
Protocol enabling real-time bidirectional communication between client and server.
WiFi
Wireless networking technology based on IEEE 802.11 standards. WiFi 6/6E is current gen.
XML
eXtensible Markup Language — human-readable data format using custom tags.
